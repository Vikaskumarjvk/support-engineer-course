# SE Academy

A beginner-friendly Support Engineer course. 24 lessons: a 15-part curriculum that takes you from zero to job-ready, plus 9 practice drill banks to build the reflexes.

Everything is generic and industry-standard. The tools are described by what they do (build system, service framework, deployment tool, ticketing, monitoring) with real-world equivalents named (Maven/Gradle, gRPC/Spring Boot, Spinnaker/Argo CD, Jira/PagerDuty, CloudWatch), so it reads the same whether you work at any company.

## What's inside

**Course (read in order)**
1. Welcome: how a tech company works
2. First days: access & where to look
3. The developer environment
4. How code lives: Git & packages
5. The build system
6. How services talk to each other
7. Infrastructure as code
8. Pipelines & deployment
9. Monitoring & alarms
10. Tickets, queues & on-call
11. Code review
12. The detective method: troubleshooting
13. Writing: COEs & runbooks
14. Leadership Principles
15. Capstone & 30/60/90 plan

**Training (do these to build the reflexes)** — troubleshooting, on-call/incident, tools fluency, behavioral/STAR, coding/scripting, operational excellence, writing, the 30/60/90 playbook, and Linux host drills.

## How to run it

It's a plain static site. No build step, no server, no dependencies.

- **Locally:** open `index.html` in any browser.
- **On the web:** push to GitHub and turn on GitHub Pages (Settings -> Pages -> Branch: `main`, folder: `/root`). It works on phone and desktop.

Your progress (which lessons you've marked done) is saved in your browser on each device.

## Files

- `index.html` — the page shell
- `assets/style.css` — styling
- `assets/app.js` — the renderer (sidebar, routing, search, progress)
- `assets/content.js` — all 24 lessons as data (auto-generated)
